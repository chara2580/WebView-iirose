根据提供的代码，MainActivity 中负责后台省电的核心逻辑主要集中在 onPause 和 onResume 生命周期中对 WebView 的暂停与恢复处理。同时包含了对音频焦点的管理。

以下是提取出的相关代码段及解析：

1. WebView 挂起与恢复 (核心省电逻辑)

这是最关键的部分。当应用进入后台时，暂停 WebView 的渲染和脚本执行，能显著降低 CPU 占用和电量消耗。

```java
@Override
protected void onPause() {
    super.onPause();
    GoNativeApplication application = getGNApplication();
    application.mBridge.onActivityPause(this);
    this.isActivityPaused = true;
    stopCheckingReadyStatus();

    // 【核心省电代码】:
    // 通过配置项 pauseWebViewOnActivityPause 控制是否在 Activity 暂停时挂起 WebView
    if (this.mWebview != null && application.mBridge.pauseWebViewOnActivityPause()) {
        this.mWebview.onPause();
    }

    // 注销网络监听，避免后台无谓的广播接收
    if (this.connectivityReceiver != null) {
        unregisterReceiver(this.connectivityReceiver);
    }

    // 持久化 Cookie，确保数据不丢失的前提下清理内存
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP) {
        CookieManager.getInstance().flush();
    }
}
```

2. WebView 恢复

当应用回到前台时，恢复 WebView 的活动状态。

```java

protected void onResume() {
    super.onResume();
    // ... 其他状态恢复 ...

    // 【恢复代码】:
    // 只有在之前确实被暂停了，才调用 onResume
    if (this.mWebview != null) this.mWebview.onResume();
    
    // ... 网络重连逻辑 ...
}
```

3. 停止后台轮询 (CPU 省电)

应用中有一个 statusChecker 的 Runnable，它会每 100ms 轮询一次 WebView 的 document.readyState。如果在后台继续运行，会导致 CPU 无法休眠。

```java
private final Runnable statusChecker = new Runnable() {
    @Override
    public void run() {
        runOnUiThread(() -> checkReadyStatus());
        handler.postDelayed(statusChecker, 100); // 0.1 sec
    }
};

// 在 onPause 中调用此方法停止轮询
private void stopCheckingReadyStatus() {
    handler.removeCallbacks(statusChecker);
}
```

4. 音频焦点管理 (针对媒体播放的省电优化)

虽然音频播放本身耗电，但在后台时合理释放或持有音频焦点是为了配合系统策略，避免与其他应用冲突导致 CPU 唤醒。

```java
// 在 onStart 中初始化音频焦点监听 (如果开启了 WebRTC 蓝牙音频)
if (AppConfig.getInstance(this).permissions.isWebRTCBluetoothAudioEnabled()) {
    AudioUtils.initAudioFocusListener(this);
}

// 提供给 Bridge 调用的接口，用于请求或放弃音频焦点
public void focusAudio(boolean enabled) {
    if (enabled) {
        AudioUtils.requestAudioFocus(this);
    } else {
        AudioUtils.abandonFocusRequest(this);
    }
}
```

5. 后台缓存清理 (内存/存储省电)

虽然主要是为了隐私和空间，但在 onStop 中清理缓存也能减少下次启动时的 IO 负担。

```java

    }
}
```
